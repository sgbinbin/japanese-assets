# 日语词条内容质量规范 v1

> 本规范定义「什么样的内容值得放进词库」。
> 每一条都有正例（✅）和反例（❌）。
> 所有词条生产者（人或 AI）必须遵守本规范。

---

## 一、例句质量标准

### 1.1 高质量例句的特征

| 标准 | 说明 |
|------|------|
| 有说话人 | 能感觉到「谁在对谁说」 |
| 有场景 | 能想象出「在什么情况下说」 |
| 有情绪 | 能感觉到说话人的心情 |
| 口语自然 | 读出来像真人会说的话 |
| 不自足 | 放在上下文中才完整，不是孤立的词典造句 |

### 1.2 正例

✅ **有场景 + 有情绪 + 有说话人**
```
「彼、いきなり反対意見言うけど、根回しが下手なんだよね。」
→ 同事之间私下聊天，带点轻蔑和无奈
```

✅ **日常真实场景**
```
「明日の面接、ちょっと不安かも。」
→ 朋友之间，带点撒娇和求安慰
```

✅ **职场真实场景**
```
「この件、部長にまだ根回ししてないの？」
→ 前辈对后辈，带点惊讶和责备
```

### 1.3 反例

❌ **词典造句感**
```
「根回しは日本文化において重要な概念である。」
→ 教科书腔，没人会这样说话
```

❌ **没有场景**
``
「彼は承認した。」
→ 谁？承认什么？在哪说的？完全不知道
``

❌ **自足型句子**
```
「影響とは、ある物事が他の物事に及ぼす作用のことです。」
→ 这是字典定义，不是例句
```

### 1.4 例句审核清单

- [ ] 能想象出谁在说吗？
- [ ] 能想象出在哪说吗？
- [ ] 读出来像真人会说的话吗？
- [ ] 有没有教科书味？
- [ ] 有没有字典味？
- [ ] 至少 2 条吗？
- [ ] 覆盖了至少 2 种语体吗（口语/书面/邮件等）？

## 二、语感解释质量标准

### 2.1 高质量语感的特征

| 标准 | 说明 |
|------|------|
| 说感觉不说定义 | 告诉读者「这个词摸起来什么质感」 |
| 用比喻 | 比抽象描述更容易理解 |
| 有对比 | 通过和近义词的差异来定位 |
| 有禁区 | 明确说「不要这样用」 |
| 有画面感 | 读完脑子里能出现一个场景 |

### 2.2 正例

✅ **用比喻说清语感**
```
「根回し」：移植大树前先绕着根松土。不是走后门，是日本职场的润滑剂。
正式决定之前，私下把各方意见对齐。不做根回し直接上会，
在日本人眼里等于「不会读空气」。
```

✅ **用对比定位语感**
```
「モメる」比「争论」轻，比「吵架」重。
核心感觉是「拖拖拉拉地来回拉扯」。
不是激烈的冲突，而是各方立场不同导致事情卡住了。
```

✅ **用画面感描述**
```
「ざわつく」：像有人在你心里窸窸窣窣地翻东西，
你说不清在翻什么，但就是静不下来。
```

### 2.3 反例

❌ **字典式定义**
```
「根回し」：事前に関係者と話し合い、了解を得ておくこと。
→ 这是字典复制粘贴，不是语感
```

❌ **空洞的形容词堆砌**
```
「承認」：这是一个很重要的正式词汇，在商务场合经常使用。
→ 说了等于没说
```

❌ **只翻译不解释**
```
「渋い」：涩的，有品位的。
→ 中文「涩」和日语「渋い」语感完全不同，必须解释
```

### 2.4 语感审核清单

- [ ] 有没有用比喻或画面？
- [ ] 有没有和近义词做对比？
- [ ] 有没有说「不要这样理解」？
- [ ] 读完能不能感觉到这个词的「温度」？
- [ ] 有没有区分口语/书面语的使用差异？
- [ ] 中国人读完会不会恍然大悟？

## 三、记忆法质量标准

### 3.1 有效记忆法的特征

| 类型 | 说明 | 有效性 |
|------|------|--------|
| 画面感 | 让你脑子里出现一个画面 | ⭐⭐⭐⭐⭐ |
| 谐音梗 | 用中文谐音挂钩 | ⭐⭐⭐⭐ |
| 词源故事 | 讲这个词怎么来的 | ⭐⭐⭐⭐ |
| 对比中文 | 和中文的异同 | ⭐⭐⭐⭐ |
| 拆字 | 把汉字拆开理解 | ⭐⭐⭐ |
| 顺口溜 | 编个口诀 | ⭐⭐⭐ |

### 3.2 有效记忆法正例

✅ **画面感**
```
「根回し」：想象一棵大树，有人蹲在树根旁边慢慢松土，
远处一群人站在会议室门口等着。
→ 画面越荒诞越记得住
```

✅ **谐音 + 画面**
```
「渋い（しぶい）」：涩柿子——不好看但越品越有味道。
→ 直接关联中文「涩」
```

✅ **词源故事**
```
「根回し」：园艺术语。移植树木前，提前一年在树冠投影范围
切断侧根，促发须根，提高存活率。
→ 知道了来源就不会忘记意思
```

✅ **中文对比**
```
「手紙」：中文是 toilet paper，日语是 letter。
→ 记住这个反差，一辈子忘不了
```

### 3.3 无效记忆法反例

❌ **废话型**
```
「承認」：承→接受，认→认可，接受认可→承认。
→ 这等于没说，中文本来就是这么写的
```

❌ **没有帮助的类比**
```
「影響」：影→影子，响→响声，影子和响声都会影响。
→ 勉强能理解，但记不住
```

❌ **重复释义型**
```
「確認」：确→确定，认→认，确认→确认。
→ 把中文翻译再说一遍，零帮助
```

❌ **过于复杂**
```
「培う」：想象一个农民在培养皿里种土豆，
培养皿代表实验室，土豆代表人才……
→ 绕了八个弯，不如直接记
```

### 3.4 记忆法审核清单

- [ ] 读完 10 秒内能形成画面吗？
- [ ] 隔天还能想起来吗？
- [ ] 有没有废话？（把中文拆一遍不算记忆法）
- [ ] 有没有过于复杂？
- [ ] 对中国人有针对性吗？
- [ ] 是不是每个词都需要记忆法？（简单词可以不写）

## 四、中国人常见理解错误分类

### 4.1 五大类错误

| 类型 | 典型案例 | 为什么会错 |
|------|----------|-----------|
| **同形异义** | 手紙≠手纸，勉強≠勉强 | 汉字相同但意思完全不同 |
| **同义异感** | 承認≈認める（语感不同） | 中文翻译一样但日语语感不同 |
| **直觉陷阱** | 渋い≠涩（贬义） | 中文直觉会给出错误判断 |
| **用法差异** | 影響を与える（不是影響を出す） | 搭配和中文不同 |
| **读音陷阱** | 今日（きょう）不是「こんにち」 | 同一个汉字有多种读法 |

### 4.2 每个词条必须覆盖的错误类型

```
同形异义词 → 必须在 chinese_cognate.trap 中标注
近义词     → 必须在 pitfalls.confused_with 中对比
搭配差异   → 必须在 usage.collocations 中列出
读音陷阱   → 必须在 pitfalls.reading_traps 中标注
语感差异   → 必须在 nuance.similar_nuance 中说明
```

### 4.3 错误示例的写法

✅ **好写法：给出具体错误场景**
```
中国人常把「根回し」翻译成「走后门」，
但日语里它是中性偏正面的。
说「彼は根回しが上手」是在夸他沟通能力强，
不是说他搞关系。
```

❌ **坏写法：只说「不要误解」**
```
注意：「根回し」不要误解。
→ 没说怎么误解的，没说正确的理解是什么
```

## 五、如何避免「字典味」

### 5.1 字典味的特征

| 特征 | 示例 |
|------|------|
| 用「…のこと」结尾 | 「影響とは、物事に作用を及ぼすこと」 |
| 用「である」体 | 「日本では重要な概念である」 |
| 定义式开头 | 「XXとは…」 |
| 无主语无场景 | 「確認する必要がある」 |
| 抽象名词堆砌 | 「社会的意義を有する」 |

### 5.2 去字典味的方法

| 方法 | 改前 | 改后 |
|------|------|------|
| 加说话人 | 「確認する必要がある」 | 「ちゃんと確認しておいてね」 |
| 加场景 | 「承認を得る」 | 「部長に承認してもらったから、進めていいよ」 |
| 加情绪 | 「不安を感じる」 | 「明日の面接、ちょっと不安かも…」 |
| 用口语 | 「重要な役割を果たす」 | 「この仕事、かなり重要なポジションだよ」 |
| 用省略 | 「〜と思われる」 | 「〜じゃないかな」 |

## 六、如何避免「教材味」

### 6.1 教材味的特征

| 特征 | 示例 |
|------|------|
| 过于礼貌 | 「田中さんは会社に行きます」 |
| 过于完整 | 「私は毎朝7時に起きます」 |
| 教学式对话 | 「A: すみません、駅はどこですか。B: あそこです」 |
| 无信息量 | 「今日はいい天気ですね」 |
| 刻意使用目标词 | 「この単語はとても重要です」 |

### 6.2 去教材味的方法

| 方法 | 改前 | 改后 |
|------|------|------|
| 加不完整感 | 「田中さんは会社に行きます」 | 「田中さん、今日会社くるかな」 |
| 加省略 | 「すみません、駅はどこですか」 | 「あのう、駅ってこの辺ですか」 |
| 加真实语气 | 「今日は暑いです」 | 「うわ、今日マジで暑い」 |
| 加上下文 | 「食べますか？」 | 「これ、もう食べないの？じゃあもらうね」 |
| 加口语缩略 | 「〜ている」 | 「〜てる」 |

### 6.3 不同场景的自然度

| 场景 | 教材风 | 真实风 |
|------|--------|--------|
| 朋友聊天 | 「映画を見に行きませんか」 | 「映画行かない？」 |
| 职场邮件 | 「ご確認をお願いいたします」 | 「確認お願いします」 |
| 抱怨 | 「この問題は難しいです」 | 「これマジで無理なんだけど」 |
| 惊讶 | 「そうですか」 | 「え、マジで？」 |

## 七、增强「真实日本生活感」

### 7.1 生活感的来源

| 来源 | 说明 |
|------|------|
| 省略 | 日本人说话极度省略，主语经常不说 |
| 语气词 | ね、よ、な、よね、でしょ |
| 口语缩略 | てる、ちゃう、じゃん、っす |
| 外来语 | マジ、ヤバい、ウケる |
| 读空气 | 不直接说，靠语气和上下文 |

### 7.2 生活感例句

✅ **有生活感**
```
「これ、もう食べないの？じゃあもらうね。」
→ 日本人真的会这样说话
```

✅ **有语气词**
```
「明日の面接、ちょっと不安かも…」
→ 「かも」加上「…」传达出不确定和犹豫
```

✅ **有省略**
```
「部長、まだ帰ってないの？」
→ 省略了「部長は」，自然
```

## 八、增强「职场感」

### 8.1 职场感的来源

| 来源 | 说明 |
|------|------|
| 上下关系 | 前辈/后辈、上司/部下的语气差异 |
| 邮件用语 | 「ご確認お願いします」「お疲れ様です」 |
| 委婉表达 | 「ちょっと難しいかも…」= 不行 |
| 空气感 | 「考えておきます」= 不会做 |
| 会议用语 | 「ここは一旦持ち帰りましょう」 |

### 8.2 职场感例句

✅ **有上下关系**
```
上司：「この件、ちょっと対応してもらえる？」
部下：「はい、すぐやります。」
→ 虽然用了「もらえる？」但实际上是命令
```

✅ **有委婉拒绝**
```
「スケジュール的にちょっと厳しいかも…」
→ 日本人说「ちょっと厳しい」就是不行
```

✅ **有读空气**
```
「考えておきますね。」
→ 字面是「我会考虑的」，实际是「不会做的」
```

## 九、增强「情绪感」

### 9.1 情绪感的来源

| 来源 | 说明 |
|------|------|
| 语气词 | よ、ね、な、わ、ぞ、ぜ |
| 省略号 | 「ちょっと…」比「ちょっと」更有情绪 |
| 感叹词 | え、うわ、わあ、ひゃあ |
| 口语强调 | マジで、超、めっちゃ |
| 身体语言提示 | （ため息）（声が震える） |

### 9.2 情绪感例句

✅ **焦虑**
```
「あ、もうそんな時間！間に合わない…」
→ 慌张感扑面而来
```

✅ **无奈**
```
「また残業か…」（ため息）
→ 叹气比文字更有感染力
```

✅ **开心**
```
「やった！合格した！」
→ 感叹号 + やった 传达兴奋
```

✅ **委婉的不满**
```
「…ちょっと、それは困るんだけど。」
→ 省略号 + けど 传达忍耐中的不满
```

## 十、AI 出题适配性

### 10.1 适合 AI 出题的内容

| 字段 | 适合的题型 |
|------|-----------|
| `meaning.meanings[].cn` | 选择题：选出正确中文意思 |
| `examples.sentences[].blank` | 填空题：补全句子 |
| `nuance.similar_nuance` | 辨析题：A 和 B 有什么区别 |
| `usage.collocations` | 搭配题：选出正确搭配 |
| `pitfalls.common_mistakes` | 改错题：下面哪句是错的 |
| `pitfalls.confused_with` | 选择题：这个词和 XX 的区别是什么 |
| `memory.chinese_cognate.trap` | 判断题：中文直觉会给出的答案是错的 |
| `relations.synonyms` | 排序题：按语感从强到弱排列 |
| `relations.antonyms` | 连线题：配对反义词 |

### 10.2 题目模板示例

**选择题模板：**
```
「{word}」最接近的中文意思是？
A. {meaning.meanings[0].cn}  ← 正确
B. {confused_with[0].word 的中文}  ← 干扰项
C. {chinese_cognate.trap 的中文直觉}  ← 中国人专属陷阱
D. 随机同级别词的中文
```

**填空题模板：**
```
{examples.sentences[0].blank}
A. {word}  ← 正确
B. {confused_with[0].word}
C. {synonyms[0].word}
D. 随机词
```

**语感辨析题模板：**
```
「{word}」和「{similar_nuance[0].word}」的区别是什么？
→ AI 生成答案基于 similar_nuance.diff
```

## 十一、Anki 卡片适配性

### 11.1 适合做 Anki 卡的内容

| 内容 | 卡片类型 | 优先级 |
|------|----------|--------|
| word → meaning | 正面卡 | P0 |
| meaning → word | 反面卡 | P0 |
| examples.blank | 填空卡 | P0 |
| collocations | 搭配卡 | P1 |
| confused_with | 辨析卡 | P1 |
| reading_traps | 读音卡 | P1 |
| nuance.feeling | 语感卡（纯阅读） | P2 |
| keigo_relation | 敬语卡 | P2 |

### 11.2 Anki 卡片生成规则

```
每个词条至少生成：
1 张 正面卡（看日文想中文）
1 张 反面卡（看中文想日文）
N 张 填空卡（N = examples.sentences 带 blank 的数量）
0~2 张 辨析卡（如果有 confused_with）
0~1 张 读音卡（如果有 reading_traps）

总计：每个词条 3~7 张卡片
```

### 11.3 Anki 审核清单

- [ ] 填空卡的 blank 位置是否合理？（不能把助词挖掉）
- [ ] 正面卡和反面卡的中文是否自然？
- [ ] 辨析卡的干扰项是否有区分度？
- [ ] 卡片数量是否在 3-7 张之间？

- [ ] 有没有针对中国人的提醒？
- [ ] 结尾有没有记忆锚点？

## 十三、内容审核总规则

### 13.1 三层审核

| 层级 | 审核内容 | 审核标准 |
|------|----------|----------|
| L1 基础 | 字段完整性 | P0 字段必须有值 |
| L2 质量 | 内容自然度 | 无字典味、无教材味 |
| L3 价值 | 教学有效性 | 记忆法有帮助、易错点有针对性 |

### 13.2 一票否决项

以下情况直接退回重写：

- ❌ 例句像字典定义
- ❌ 记忆法是废话（把中文拆一遍）
- ❌ 语感解释只是翻译
- ❌ 易错点只说「注意」不说具体
- ❌ 例句没有场景
- ❌ 中文翻译有 AI 味（过于正式、不自然）
- ❌ 有重复词条

### 13.3 质量评分（每条 1-5 分）

| 维度 | 权重 | 说明 |
|------|------|------|
| 例句自然度 | 25% | 像真人说话吗？ |
| 语感深度 | 25% | 有没有超越字典？ |
| 记忆有效性 | 20% | 10秒内能形成画面吗？ |
| 易错点针对性 | 15% | 对中国人有帮助吗？ |
| 文化丰富度 | 10% | 有没有文化背景？ |
| AI 出题适配 | 5% | 模板能不能直接用？ |

**合格线：加权平均 ≥ 3.5 分**

### 13.4 AI 生成内容的额外审核

AI 生成的内容容易出现以下问题，需要重点检查：

| 问题 | 检查方法 |
|------|----------|
| 过于工整 | 每个词条结构完全一样 → 打破模板 |
| 过于安全 | 只用常见词 → 加入风险词 |
| 过于笼统 | 语感描述模糊 → 要求具体画面 |
| 中文不自然 | 翻译腔 → 让不懂日语的人读中文 |
| 记忆法雷同 | 每个词都用谐音 → 要求类型多样性 |
